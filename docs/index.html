Live network telemetry · in 3D

Your network traffic
as a night highway.

PacketRush captures your machine's packets in real time and drives them down a 3D motorway. Every packet is a vehicle — its type is the protocol, its size is the payload, its direction is inbound vs. outbound. A download becomes a convoy of trucks.

Packet size → vehicle scale

Bigger packets drive bigger vehicles. The scale ramps linearly with payload and clamps at one full MTU, so a 1500-byte frame is ~55% larger than a 60-byte ACK.

scale = 0.8 + min(len / 1500, 1) × 0.45

Under the hood

From kernel to canvas in ~150 ms

One small Node server reads tcpdump, parses and classifies every line, tags flows, resolves hostnames, and streams 100 ms batches over WebSocket. The browser renders the highway with Three.js.

01 · PARSE

tcpdump lines

IPv4/IPv6, ports, length, ARP. Malformed lines drop to null. Pure functions, unit-tested.

02 · FLOW

Convoys

A normalized 5-tuple table (30s idle eviction) groups a connection's packets into one lane.

03 · RENDER

Two draw calls / type

InstancedMesh keeps 1000 vehicles at 60fps. A fixed bounding sphere keeps raycast picking alive.

Beyond the pretty picture

It's an inspector, not just a toy

Click any vehicle to read the packet behind it. Filter protocols, watch convoys form, read where your traffic is headed — all from a HUD that streams live.

Click to inspect

Every vehicle is a real packet

Raycast picking shows the hostname (server-side cached reverse-DNS), ports, protocol family and byte size — with a pulsing selection ring that tracks the vehicle. Click empty road to dismiss.

GeoIP exit signs

See where your traffic is going

A bundled offline prefix table maps remote IPs to org + country. The busiest destinations sprout green roadside exit signs — Google, Cloudflare, Fastly, AWS — entirely offline, even in demo mode.

Live HUD + sparkline

Throughput at a glance

Packets/sec in and out, KB/s bandwidth, active vehicle count, and a 60-second sparkline of rate and bandwidth. Plus protocol filters (click a legend chip to mute, alt-click to solo) and an optional synth engine hum that follows traffic intensity.

Built to scale

1000 vehicles, still 60fps

Instanced rendering means two draw calls per vehicle type no matter how busy the wire gets. A bounded flow table, spawn queue and rDNS cache keep memory flat under a flood.

Get rolling

Up and running in three commands

No build step, no framework, no bundler. Node 18+, and one dependency.

# clone & install
git clone https://github.com/MeyerThorsten/PacketRush
cd PacketRush && npm install

# run — sudo for live capture on macOS
sudo npm start

# open
open http://localhost:8090

No root? It still runs — the page falls back to a simulated traffic stream so you can explore every scene. On Linux, detect the route with ip route or grant cap_net_raw to tcpdump for sudo-free capture.

Configuration

PORT=9000 — change the web port
IFACE=en1 — capture a specific interface
?theme=ocean — start in any scene
?demo&pps=900 — stress the demo generator

Verify

npm test      # 31 unit tests
npm run check # syntax-check modules

Full reference in DOCUMENTATION.md — architecture, capture privileges, and how to author your own scene or fleet.
